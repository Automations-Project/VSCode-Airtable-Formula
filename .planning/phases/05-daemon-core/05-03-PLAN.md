const requireBearer = (req, res, next) => {
  const header = req.headers?.authorization ?? '';
  const match = header.match(/^Bearer\s+(.+)$/i);
  const provided = match ? match[1] : null;
  if (provided !== currentToken.bearerToken) {
    res.status(401).json({ error: 'Unauthorized' });
    return;
  }
  next();
};

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → daemon HTTP server (loopback) | Untrusted callers on localhost can probe the daemon; bearer auth required on all endpoints |
| daemon HTTP server → AirtableAuth/AirtableClient | Auth credentials flow through daemon to Airtable's internal API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Elevation of Privilege | POST /mcp | mitigate | `requireBearer` middleware applied to `/mcp` — unauthenticated callers receive 401 |
| T-05-03-02 | Information Disclosure | GET /daemon/health | mitigate | Health response omits bearer token; always gated by bearer auth |
| T-05-03-03 | Information Disclosure | bearer token in logs | mitigate | `currentToken.bearerToken` must never appear in any console.log/warn/error; log only pid and port |
| T-05-03-04 | Denial of Service | WHATWG blocked port | mitigate | `listenAvoidingBlockedPorts` retries up to 5 times when port 0 produces a blocked port |
| T-05-03-05 | Denial of Service | unauthenticated shutdown | mitigate | `requireBearer` applied before shutdown logic — unauthenticated callers cannot terminate the daemon |
| T-05-03-06 | Spoofing | stale bearer after rotation | mitigate | `get bearerToken()` live getter reads from `currentToken` — rotation immediately invalidates old tokens |
</threat_model>
