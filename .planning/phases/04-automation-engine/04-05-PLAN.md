## Goal

Create all four static asset files needed for AUTO-01 and AUTO-05. These files have no code dependencies and can be created in parallel with Plans 04-03 and 04-04. Plan 04-07 (the wiring plan) references all four files from `package.json`.

## Tasks

### Task 1: Create grammar and language configuration

**Files:**
- `packages/extension/syntaxes/airtable-automation.tmLanguage.json`
- `packages/extension/language-configuration/airtable-automation-language-configuration.json`

**Action:**

Create `packages/extension/syntaxes/airtable-automation.tmLanguage.json`. This is a direct copy of `packages/extension/syntaxes/airtable-script.tmLanguage.json` (9 lines) with three field substitutions:

- `"scopeName"`: `"source.airtable-script"` becomes `"source.airtable-automation"`
- `"name"`: `"Airtable Script"` becomes `"Airtable Automation"`
- `"fileTypes"`: `["script", "ats"]` becomes `["automation", "ata"]`

The `patterns` array with `{ "include": "source.js" }` is copied verbatim — this is the JS syntax embedding pattern.

Full file content to write:
```json
{
  "$schema": "https://raw.githubusercontent.com/martinring/tmlanguage/master/tmlanguage.json",
  "scopeName": "source.airtable-automation",
  "name": "Airtable Automation",
  "fileTypes": ["automation", "ata"],
  "patterns": [
    { "include": "source.js" }
  ]
}
```

Create `packages/extension/language-configuration/airtable-automation-language-configuration.json`. This is a verbatim copy of `packages/extension/language-configuration/airtable-script-language-configuration.json` (92 lines). No content changes needed — the script language configuration already has the correct JS settings:
- Line/block comment style (`//` and `/* */`)
- JS identifier word pattern (`[a-zA-Z_$][a-zA-Z0-9_$]*`)
- JS bracket pairs and indentation rules
- Backtick auto-close (important for template literals in automation scripts)

Read the file at `packages/extension/language-configuration/airtable-script-language-configuration.json` and copy its full content without modification.

**Verify:**

Both JSON files parse without error:
```
node -e "JSON.parse(require('fs').readFileSync('packages/extension/syntaxes/airtable-automation.tmLanguage.json', 'utf8'))" && echo "grammar OK"
node -e "JSON.parse(require('fs').readFileSync('packages/extension/language-configuration/airtable-automation-language-configuration.json', 'utf8'))" && echo "language-config OK"
```

Expected: "grammar OK" and "language-config OK".

**Done:** Both JSON files exist and parse correctly.

### Task 2: Create SVG file icons

**Files:**
- `packages/extension/icons/automation-light.svg`
- `packages/extension/icons/automation-dark.svg`

**Action:**

Create two SVG icon files. These follow the exact pattern established by `packages/extension/icons/script-light.svg` and `script-dark.svg` — same dimensions (16x16), same rounded rectangle, same font settings — with two changes: letter `A` instead of `S`, and file-specific fill colors.

**automation-light.svg** — use the light-theme fill color `#388E3C` (same as `script-light.svg`):

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 16 16">
  <rect width="16" height="16" rx="2" fill="#388E3C"/>
  <text x="8" y="12" font-size="10" font-family="monospace" fill="white" text-anchor="middle">A</text>
</svg>
```

**automation-dark.svg** — use the dark-theme fill color `#2E7D32` (same as `script-dark.svg`):

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 16 16">
  <rect width="16" height="16" rx="2" fill="#2E7D32"/>
  <text x="8" y="12" font-size="10" font-family="monospace" fill="white" text-anchor="middle">A</text>
</svg>
```

The green color family is used for all Airtable-specific language file types (formula, script, automation) to visually group them in the file explorer.

**Verify:**

Both files exist:
```
ls packages/extension/icons/automation-light.svg packages/extension/icons/automation-dark.svg
```

Both files contain the letter `A` (not `S`):
```
grep -l "A</text>" packages/extension/icons/automation-light.svg packages/extension/icons/automation-dark.svg
```

Expected: both files listed.

**Done:** Both SVG icon files exist with letter `A` and correct green fills.
