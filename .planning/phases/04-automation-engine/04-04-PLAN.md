## Goal

Implement `automationDiagnostics` — the wrong-context-only diagnostic engine for `.automation` files. This file is simpler than `script/diagnostics.ts` because it omits `checkMissingAwait` and `checkUnknownGlobals` entirely (D-04, D-05). The only novel code is `checkWrongContext` with its 15 forbidden patterns.

Critical implementation requirement: every `/g` regex used in a scan loop **must** have `pattern.lastIndex = 0` reset at the start of that loop iteration. Module-scope `/g` patterns retain `lastIndex` state between calls — failure to reset causes the second document scan to silently miss matches. The sequential-call regression test in `diagnostics.test.ts` catches this if omitted.

**Step 4 — Implement checkWrongContext:**

```typescript
function checkWrongContext(
    text: string,
    exclusionRanges: Array<{ start: number; end: number }>
): LsDiagnostic[] {
    const diagnostics: LsDiagnostic[] = [];

    for (const entry of FORBIDDEN_PATTERNS) {
        // REQUIRED: reset module-scope /g pattern before each scan.
        // Without this reset, the second call on any document skips matches
        // before the stale lastIndex position (RESEARCH.md Pitfall 1).
        entry.pattern.lastIndex = 0;
        let match: RegExpExecArray | null;
        while ((match = entry.pattern.exec(text)) !== null) {
            if (isInsideExclusionRange(match.index, exclusionRanges)) continue;
            diagnostics.push({
                range: makeRange(text, match.index, match.index + match[0].length),
                message: entry.message,
                severity: LsSeverity.Warning,
                code: 'wrong-context',
                source: 'airtable-automation',
            });
        }
    }

    return diagnostics;
}
```

Note: `entry.pattern.exec(text)` is the standard RegExp method for iterating global pattern matches in a while loop. This is the same pattern used in `script/diagnostics.ts` for all its scan loops.

**Step 5 — Implement the public export (D-04 shape):**

```typescript
export function automationDiagnostics(text: string, _uri?: string): LsDiagnostic[] {
    const exclusionRanges = getExclusionRanges(text);
    return checkWrongContext(text, exclusionRanges);
}
```
