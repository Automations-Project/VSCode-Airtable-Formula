## Component Inventory

All components reuse existing CSS classes from `packages/webview/src/styles.css`. No new classes required unless noted.

### Tunnel Section — glass-panel

Structure follows the exact pattern of existing Setup tab glass-panels:

```
<div class="glass-panel">
  <div class="section-header">
    <div class="eyebrow">Remote Access</div>
    <div class="title">Tunnel</div>
    <div class="detail">{status line}</div>
  </div>
  {tunnel body}
</div>
```

### Provider Picker

Use existing `.select-input` class (already styled in `styles.css`).
Options: "Cloudflare Quick Tunnel", "ngrok", "Cloudflare Named Tunnel".
Provider IDs: `cf-quick`, `ngrok`, `cf-named`.

### Tunnel URL Display

Use `.font-mono` style (`font-family: var(--font-mono); font-size: 0.7rem`).
Wrap in a `.list-row` with a copy-to-clipboard icon button (`.btn .btn-ghost .btn-sm`).
Show only when `tunnelState.url` is present.

### ngrok Authtoken Input

Rendered when `provider === 'ngrok'` AND no authtoken is stored.
Standard `<input type="password">` styled as:
```css
font-size: 0.7rem;
font-family: var(--font-mono);
background: var(--bg-input);
border: 1px solid var(--border);
border-radius: var(--radius-md);
padding: 5px 10px;
color: var(--fg);
width: 100%;
```
On focus: `border-color: var(--at-blue)`.
Label: "ngrok Auth Token" (eyebrow style, `0.6rem`, uppercase).
Placeholder: "paste token from ngrok dashboard".

### ngrok Optional Domain Input

Rendered below authtoken when `provider === 'ngrok'`.
Same styling as authtoken input but `type="text"`.
Label: "Reserved Domain (optional)".
Placeholder: "yourname.ngrok-free.app".

### Enable/Disable Button

- **Tunnel disabled:** `.btn .btn-primary` — label "Enable Tunnel"
- **Tunnel active:** `.btn .btn-ghost` — label "Disable Tunnel"
- **Pending (starting/stopping):** disabled + opacity 0.6 + label "Starting..." / "Stopping..."

### Status Chip

Use existing `.chip` variants:
- Active: `.chip .chip-ok` — "Active"
- Disabled: `.chip .chip-muted` — "Disabled"
- Starting: `.chip .chip-info` — "Starting..."
- Auto-disabled: `.chip .chip-warn` — "Auto-disabled"

### 401-Burst Warning Banner

Rendered when `tunnelState.autoDisabledReason` is set.
Use `.bg-warn` background, `.fg-warn` text, `.border` border with `border-radius: var(--radius-md)`.
No new CSS class — inline style with existing tokens:
```
background: var(--bg-warn);
color: var(--fg-warn);
border: 1px solid rgba(255,186,5,0.2);
border-radius: var(--radius-md);
padding: 10px 12px;
font-size: 0.72rem;
```
Content: "Tunnel auto-disabled: {failures} auth failures from {ip} in {windowMs/1000}s. Re-enable when ready."

## Interaction Contract

### Tunnel Enable Flow

1. User selects provider in picker (default: `cf-quick`).
2. If `ngrok` selected and no authtoken stored: authtoken input appears (unmask on focus-visible).
3. User clicks "Enable Tunnel" button.
4. Button becomes disabled, label becomes "Starting...", chip becomes `.chip-info "Starting..."`.
5. Webview sends `{ type: 'tunnel:enable', provider, authtoken?, domain? }` to extension.
6. Extension calls `POST /daemon/enable-tunnel`.
7. On `daemon:tunnel-started` SSE: chip → `.chip-ok "Active"`, URL row appears, button → `.btn-ghost "Disable Tunnel"`.
8. On error: chip → `.chip-err`, button re-enables with "Enable Tunnel".

### Tunnel Disable Flow

1. User clicks "Disable Tunnel" button.
2. Button becomes disabled, label → "Stopping...", chip → `.chip-info "Stopping..."`.
3. Webview sends `{ type: 'tunnel:disable' }`.
4. Extension calls `POST /daemon/disable-tunnel` (or `airtableFormula.tunnel.disable` command).
5. On `daemon:tunnel-stopped` SSE: chip → `.chip-muted "Disabled"`, URL row hidden, button → `.btn-primary "Enable Tunnel"`.

### 401-Burst Auto-Disable

1. Extension receives `daemon:tunnel-auto-disabled` SSE.
2. DashboardState updated: `tunnel.status = 'auto-disabled'`, `tunnel.autoDisabledReason = { failures, windowMs, ip }`.
3. Warning banner appears below section-header.
4. Chip → `.chip-warn "Auto-disabled"`.
5. Button returns to "Enable Tunnel" (active, not disabled) — user can re-enable immediately.

### Provider Switch

- Provider picker is always enabled when tunnel is disabled.
- Provider picker is disabled (opacity 0.6, `pointer-events: none`) when tunnel is active (must disable first).
- Switching provider clears the stored authtoken input display (does not clear SecretStorage until user saves new value).

### State Persistence

- Provider selection and enabled state are shown from `DashboardState.tunnel` (read from `tunnel-settings.json` via lockfile/extension).
- Webview never writes directly to `tunnel-settings.json` — all state changes go through extension messages.

## Accessibility

- All interactive elements have visible focus ring: `outline: 2px solid var(--at-blue); outline-offset: 2px` (matches existing `.action-card:focus-visible` pattern).
- Authtoken input: `type="password"` with visible label above, `aria-describedby` pointing to helper text.
- Enable/Disable button: `aria-busy="true"` while loading.
- Warning banner: `role="alert"` so screen readers announce it when it appears.
- Provider picker: `<label>` element associated via `htmlFor`.
- Copy button: `aria-label="Copy tunnel URL"`, transient `aria-live="polite"` region for "Copied!" feedback.
