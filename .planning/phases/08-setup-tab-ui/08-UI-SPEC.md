## Spacing Scale

All spacing uses the existing `stack` utility classes and inline `style` props consistent with the rest of Setup.tsx. Values derive from the established 4-point grid already in use.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Chip internal gap, icon-to-label gap within list-row |
| sm | 8px | `stack stack-sm` gap; gap between sub-tab variants; margin-bottom inside glass-panel sections |
| md | 12px | `glass-panel` internal padding (established by `.glass-panel { padding: 12px }`) — approved project addition to standard 4-point grid (12 = 3×4; already used throughout existing codebase as the `md` glass-panel padding token) |
| lg | 16px | `stack stack-lg` gap — outer gap between glass-panel blocks (tab-level stack) |
| xl | 24px | Empty state padding (`padding: '24px 12px'` — existing pattern from Setup.tsx line 284) |

Read-only inherited exceptions (do NOT change these values in implementations — they come from existing `.section-header` and `.list-row` rules in `styles.css`):
- `section-header` margin-bottom is `10px` — inherited from `styles.css` — do not change
- List-row padding is `10px 12px` — inherited from `styles.css` — do not change
- Glass-panel border-radius is `14px` — existing token — do not change

New spacing values introduced by this spec (all multiples of 4):
- Copy button absolute positioning: `top: 8, right: 8`
- Inner variant sub-tab padding: `4px 8px`
- Outer IDE tab button padding: `8px 12px`
- Pre block padding: `8px 12px`
- Tab button `marginBottom: 0` — the 1px flush between active tab underline and the panel border-bottom is achieved by the container's `borderBottom` being on the wrapping div, not by negative margin on the tab button

## Typography

All sizes sourced from existing `styles.css` type scale — do not introduce new font sizes. Maximum 4 declared sizes, maximum 2 declared weights.

| Role | Size | Weight | Line Height | CSS |
|------|------|--------|-------------|-----|
| Section title | 0.85rem | 600 | 1.2 | `.section-header .title` |
| Body / row labels / field labels / sub-tab labels | 0.7rem | 500 (inactive) / 600 (active) | 1.4 | `fontSize: '0.7rem'` |
| Monospace (snippet code) | 0.7rem | — (inherits base) | 1.5 | `fontFamily: 'var(--font-mono)', fontSize: '0.7rem'` |
| Eyebrow / metric label | 0.6rem | 500 | 1 | `.eyebrow`, `.metric-label` |

Declared weights: `500` (inactive tab labels, muted row labels, eyebrow) and `600` (active tab labels, section titles). Body text that does not need explicit weight specification inherits from the base stylesheet — do not redeclare a third weight in component code.

Size consolidation notes:
- `0.7rem` is used for: field labels, section labels, body row text, inner variant sub-tab labels, and monospace code blocks. A single value covers all label-scale roles.
- `0.72rem` (previously listed for body) is merged into `0.7rem` — the 0.02rem difference is not perceptible and eliminated to keep the scale at 4 sizes.
- `0.65rem` (previously listed for inner sub-tab) is replaced with `0.7rem` — sub-tabs use the same label size as other labels.

Body base: 13px (set on `body` in styles.css). Line height for running text: 1.4 (matches `.section-header .detail { line-height: 1.4 }`).

## Color

All color values sourced from `styles.css` CSS custom properties. Do not use raw hex values in new JSX — always use `var(--token)`.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--bg` | `var(--at-gray800)` = rgb(29,31,37) | Tab background surface |
| Secondary (30%) | `--bg-inset` / glass-panel | `var(--at-gray900)` = rgb(17,18,21) / gradient overlay | glass-panel, list-row, snippet code blocks |
| Accent (10%) | `--at-blue` | rgb(22,110,225) | Active IDE tab underline, `btn-primary` background, `--border-focus` |
| Healthy (semantic) | `chip-ok` | `rgba(4,138,14,0.14)` bg + `var(--fg-ok)` text | Health chip when daemon is healthy |
| Degraded (semantic) | `chip-warn` | `rgba(255,186,5,0.16)` bg + `var(--fg-warn)` text | Health chip when daemon is degraded |
| Destructive | `--fg-err` / `chip-err` | `var(--at-redLight1)` text | Not used in this phase (no destructive actions) |

Accent reserved for:
- Active IDE tab indicator underline (2px `--at-blue` bottom border, matching App.tsx tab pattern)
- `btn-primary` (not used in this phase — copy button is ghost style)
- Active sub-tab indicator underline within snippet panels

Accent is NOT used for: health chip, uptime display, port number text, code block backgrounds.

### Primary Visual Anchors Per Panel

- Daemon status block: the health chip (`chip-ok` / `chip-warn`) is the primary focal point — it communicates go/no-go at a glance before the user reads any label
- MCP Config Snippets panel: the outer IDE tab bar is the primary focal point — it is the first interactive element and gates all content below
- LSP Config Snippets panel: same as MCP — the outer IDE tab bar is the primary focal point

### Snippet Code Block Background

Use `var(--bg-input)` (`rgba(0,0,0,0.25)`) for the `<pre>` background. This matches the existing input field style and creates clear visual separation from the glass-panel surface.

### Tab Component Colors

IDE tab selector (outer) and variant sub-tab (inner) share the same pattern as App.tsx tab navigation:
- Inactive tab: `color: var(--fg-muted)`, no border, `fontWeight: 500`
- Active tab: `color: var(--fg)`, `borderBottom: '2px solid var(--at-blue)'`, `fontWeight: 600`
- Tab bar border-bottom: `1px solid var(--border)`

### 1. DaemonStatusInfo Interface (packages/shared/src/types.ts)

```typescript
export interface DaemonStatusInfo {

  port_lsp:   number | null;   // LSP TCP port — null when LSP not started
```

### 2. Daemon Status Block (new — first panel in Setup.tsx)

**Visibility rule (D-01):** Render only when `daemon?.running === true`. No offline placeholder.

**Primary visual anchor:** the health chip — always rendered first within the panel body so it is the first element the user sees after the section header.

```
<div className="glass-panel">           ← identical to all other panels
  <div className="section-header">
    <div className="eyebrow">Daemon</div>
    <div className="title">MCP Server Status</div>
    <div className="detail">Local daemon running on this machine</div>
  </div>

  {/* Health chip — always shown when block is visible */}
  <div style={{ marginBottom: 8 }}>
    {daemon.healthy
      ? <span className="chip chip-ok">Healthy</span>
      : <span className="chip chip-warn">Degraded</span>}
  </div>

  {/* Key-value rows */}
  <div className="stack stack-sm">
    <div className="list-row">                          ← MCP port — always shown
      <span style={{ fontSize: '0.7rem', color: 'var(--fg-muted)', flex: 1 }}>MCP Port</span>
      <span style={{ fontFamily: 'var(--font-mono)', fontSize: '0.7rem' }}>{daemon.port}</span>
    </div>

    {daemon.port_lsp !== null && (                      ← LSP port — hidden when null (D-02)
      <div className="list-row">
        <span style={{ fontSize: '0.7rem', color: 'var(--fg-muted)', flex: 1 }}>LSP Port</span>
        <span style={{ fontFamily: 'var(--font-mono)', fontSize: '0.7rem' }}>{daemon.port_lsp}</span>
      </div>
    )}

    {daemon.tunnelUrl && (                              ← Tunnel URL — hidden when null (D-02)
      <div className="list-row">
        <span style={{ fontSize: '0.7rem', color: 'var(--fg-muted)', flex: 1 }}>Tunnel URL</span>
        <span style={{ fontFamily: 'var(--font-mono)', fontSize: '0.7rem',
          overflow: 'hidden', textOverflow: 'ellipsis', whiteSpace: 'nowrap', maxWidth: '60%' }}>
          {daemon.tunnelUrl}
        </span>
      </div>
    )}

    <div className="list-row">                          ← Uptime — always shown (D-04 format)
      <span style={{ fontSize: '0.7rem', color: 'var(--fg-muted)', flex: 1 }}>Uptime</span>
      <span style={{ fontSize: '0.7rem' }}>{formatUptime(daemon.uptime)}</span>
    </div>
  </div>
</div>
```

**Uptime format (D-04 — implementer discretion, this spec standardizes):**
Use elapsed format. Function `formatUptime(ms: number | null): string`:
- `null` → `"—"`
- `< 60_000` → `"< 1m"`
- `< 3_600_000` → `"{N}m {S}s"` (e.g., `"45m 12s"`)
- `>= 3_600_000` → `"{H}h {M}m"` (e.g., `"2h 15m"`)

### 3. Snippet Tab Component (inline in Setup.tsx — no separate file needed)

No external tab library. Use `React.useState<string>` for active tab tracking. Two independent tab states per snippet section: outer (IDE selector) and inner (variant selector).

**Primary visual anchor:** the outer IDE tab bar is rendered immediately after the section header and is the first interactive element — it gates which snippet content is shown.

**Outer IDE tab bar pattern:**
```
<div style={{
  display: 'flex', borderBottom: '1px solid var(--border)',
  marginBottom: 8, gap: 0
}}>
  {IDE_TABS.map(ide => (
    <button
      key={ide.id}
      onClick={() => setActiveIde(ide.id)}

        padding: '8px 12px', fontSize: '0.7rem',
        fontWeight: activeIde === ide.id ? 600 : 500,
        color: activeIde === ide.id ? 'var(--fg)' : 'var(--fg-muted)',
        borderBottom: `2px solid ${activeIde === ide.id ? 'var(--at-blue)' : 'transparent'}`,
        background: 'none', border: 'none', borderBottomStyle: 'solid',
        borderBottomWidth: 2, cursor: 'pointer', whiteSpace: 'nowrap',
        transition: 'color 120ms ease, border-color 120ms ease',
        marginBottom: 0,
        // The 2px active underline aligns flush with the container's 1px borderBottom.
        // No negative margin needed — the container borderBottom sits on the wrapping div.
      }}
    >{ide.label}</button>
  ))}
</div>
```

**Inner variant sub-tab bar pattern (HTTP|stdio or TCP|stdio):**
Same pattern, with tighter padding and the same label font size:

## Interaction Contracts

### Copy-to-Clipboard

Pattern: identical to `handleCopyUrl` / `setCopiedUrl` already in Setup.tsx.

```typescript
// One copied-state map covering all snippet copy targets
const [copiedKeys, setCopiedKeys] = React.useState<Record<string, boolean>>({});
const handleCopySnippet = (text: string, key: string) => {
  navigator.clipboard.writeText(text).catch(() => undefined);
  setCopiedKeys(k => ({ ...k, [key]: true }));
  setTimeout(() => setCopiedKeys(k => ({ ...k, [key]: false })), 1500);
};
```

Key format: `{section}-{ide}-{variant}` e.g., `mcp-claude-code-http`, `lsp-neovim-tcp`.

### Tab State

Two independent tab state pairs in Setup.tsx local state:

```typescript
const [mcpActiveIde, setMcpActiveIde] = React.useState('claude-code');
const [mcpActiveVariant, setMcpActiveVariant] = React.useState<'http' | 'stdio'>('http');
const [lspActiveIde, setLspActiveIde] = React.useState('claude-code');
const [lspActiveVariant, setLspActiveVariant] = React.useState<'tcp' | 'stdio'>('tcp');
```

No persistence (React state only). Tab resets to defaults on unmount/remount — this is acceptable.

### Snippet Content Computation

Snippets are computed inline (no extension round-trip, no store action). Port values are read from `useStore()` via `daemon?.port` and `daemon?.port_lsp`. Content is deterministic from state — no async computation.

```typescript
const { daemon } = useStore();
const mcpPort = daemon?.port ?? '{MCP_PORT}';
const lspPort = daemon?.port_lsp ?? '{LSP_PORT}';
```

## Accessibility

- All tab buttons: `role="tab"`, `aria-selected={isActive}`, matching App.tsx tab pattern
- Tab panel containers: `role="tabpanel"` wrapping the active snippet block
- Copy buttons: `aria-label="Copy snippet"` (generic) or `aria-label=\`Copy ${ide} ${variant} snippet\``
- `<pre>` blocks: no special ARIA needed; screen reader will read code content inline
- Health chip: no `role="status"` needed — status is part of a visual panel, not a live region. Uptime is not a live region (polling update frequency is low)
- No focus traps introduced; tab key navigates naturally through all interactive elements
