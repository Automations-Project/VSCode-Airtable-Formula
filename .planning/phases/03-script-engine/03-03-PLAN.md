## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user document text → scriptCompletions() | Text slice operations on user document content to determine completion level |
| user document text → scriptHover() | Windowed regex search on user document content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | completions.ts dot-trigger regex | mitigate | Use end-anchored regex `/\b([a-zA-Z_$][a-zA-Z0-9_$]*)\.\s*$/` applied to textToCursor slice only. Linear character-class repetitions, no nested quantifiers, end-anchored. O(n) execution bounded by cursor offset, not full document size. |
| T-03-01 | Denial of Service | hover.ts window search | mitigate | Apply dot-pattern regex to a fixed 80-character window (offset ± 40 chars), not the full document text. Regex uses two simple character-class repetitions with no alternation. O(1) bounded execution. |
| T-03-02 | Information Disclosure | hover/completions documentation values | accept | Documentation values come from SCRIPT_GLOBALS (static registry data, public Airtable API docs). No user data reflected back. No secret leakage possible. |
</threat_model>
